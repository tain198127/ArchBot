## 二、核心接口

### 2.1 Tauri Command

```rust
#[tauri::command]
async fn agent_execute_turn(
    runtime: String,          // "claude_code" | "hermes" | "opencode" | "openclaw"
    workspace_root: String,   // 项目根目录
    user_message: String,     // 用户的自然语言指令
    context_files: Vec<String>, // 当前上下文文件路径列表
) -> Result<TurnResult, String>
```

### 2.2 输入

```rust
pub struct TurnConfig {
    pub runtime: String,              // Runtime 类型
    pub workspace_root: String,       // 项目根目录
    pub user_message: String,         // 用户指令
    pub context_files: Vec<String>,   // 上下文文件
    pub git_user_name: Option<String>,
    pub git_user_email: Option<String>,
}
```

### 2.3 输出

```rust
pub struct TurnResult {
    pub turn_id: String,
    pub runtime: String,
    pub status: TurnStatus,
    pub stdout_summary: String,       // stdout 前 1000 字符（预览用）
    pub result_md_path: Option<String>, // result.md 路径
    pub file_changes: Vec<FileChange>,
    pub audit_violations: Vec<String>,  // 审计违规描述
    pub duration_ms: u64,
}

pub enum TurnStatus {
    Completed,
    Failed { reason: String },
    Timeout,
}

pub struct FileChange {
    pub path: String,
    pub change_type: String, // "created" | "modified" | "deleted"
}
```

## 三、执行流程

```
agent_execute_turn("claude_code", "/project", "分析 UserController", ["src/UserController.java"])
│
├─ 1. 加载配置
│   runtime_config::load_runtimes_config()
│   → 找到 claude_code 段
│   → build_launch_config("claude_code", entry, workspace_root, git_name, git_email)
│   → 得到 RuntimeLaunchConfig { executable, isolated_home, allowed_env, args, ... }
│
├─ 2. 获取 API token
│   secret::SecretManager::new(&machine_id)
│   → resolve("secret://claude_code/api_token")
│   → 注入到 allowed_env: ANTHROPIC_AUTH_TOKEN=sk-xxx
│
├─ 3. 初始化隔离 HOME
│   home_setup::setup_isolated_home(&IsolatedHomeConfig {
│       home_path: ~/.archbot/runtimes/claude-code/home,
│       needs_git: true,
│       ...
│   })
│
├─ 4. 生成输入文件
│   turn_dir = .archbot/agent/sessions/{uuid}/turns/{uuid}/
│   ├── 生成 input.yml（结构化输入）
│   └── 生成 prompt.txt（启动指令）
│
├─ 5. 构建启动参数
│   args = [
│     "--input-file", "{turn_dir}/input.yml",
│     "--prompt-file", "{turn_dir}/prompt.txt",
│     "--output-dir", "{turn_dir}/",
│     "--max-turns", "1",
│     "--no-interactive",
│     "--json-output"
│   ]
│
├─ 6. 启动 Runtime 子进程
│   launcher::launch_isolated_runtime(&launch_config)
│   → 得到 std::process::Child
│
├─ 7. 等待进程退出（带超时）
│   wait_with_timeout(child, timeout_seconds)
│   → stdout: NDJSON 事件流
│   → stderr: 错误日志
│
├─ 8. 扫描输出文件
│   turn_dir/result.md → 读入 result_md_path
│   turn_dir/file_changes.json → 解析 FileChange 列表
│
├─ 9. 文件访问审计
│   audit::AuditManager::audit(&accessed_paths)
│   → Vec<AuditViolation>
│
└─ 10. 返回 TurnResult
```

## 四、input.yml 生成逻辑

根据 `TurnConfig` 生成结构化输入文件（对齐架构结论 §13）：

```rust
fn generate_input_yml(config: &TurnConfig, turn_dir: &Path) -> String {
    format!(r#"
schema_version: "agent_turn_input.v1"

session_header:
  session_id: "{session_id}"
  title: "ArchBot Agent Turn"
  goal: "{user_message}"

current_turn:
  turn_id: "{turn_id}"
  user_message: "{user_message}"
  expected_output: "根据用户指令分析代码并输出结果"

recent_messages:
  - role: "user"
    content: "{user_message}"

working_context:
  project_root: "{workspace_root}"
  relevant_files:
{file_list}

execution_policy:
  runtime: "{runtime}"
  mode: "interactive_turn"
  allow_file_write: true
  allow_shell: false
  timeout_seconds: 1800

output_contract:
  stream:
    enabled: true
  final_result:
    markdown_file: "result.md"
    json_file: "result.json"
  file_changes:
    enabled: true
    output_file: "file_changes.json"
"#,
        session_id = uuid::Uuid::new_v4(),
        turn_id = uuid::Uuid::new_v4(),
        user_message = config.user_message,
        workspace_root = config.workspace_root,
        file_list = config.context_files.iter()
            .map(|f| format!("    - path: \"{}\"", f))
            .collect::<Vec<_>>().join("\n"),
        runtime = config.runtime,
    )
}
```

## 五、prompt.txt 生成逻辑

对齐架构结论 §14：

```text
请读取以下结构化输入文件，并严格根据其中内容执行本轮任务：

{turn_dir}/input.yml

执行要求：
1. 以 input.yml 作为本轮真实输入，不要只根据本 prompt 执行
2. 遵守 input.yml 中的 working_context 和 execution_policy
3. 最终结果写入 result.md 和 result.json
4. 如果产生文件变更，写入 file_changes.json
```

## 七、验证方案

实现后，验证 Claude Code 集成成功的步骤：

```
1. cargo build → 确保编译通过

2. 确保 Claude Code 可执行文件存在:
   ls ~/.archbot/runtimes/claude-code/current/claude

3. 确保 API token 已配置:
   (通过 Settings UI 或 CLI 写入)
   secret_manager.store("claude_code", "api_token", "sk-xxx")

4. 在 ArchBot 中打开一个项目 → 右键任意文件 → "分析这个文件"

5. 预期结果:
   - OnboardingWizard 首次启动展示进度
   - 右键菜单显示"分析"选项
   - 点击后 BottomPanel 的 log tab 显示 turn.delta 事件流
   - turn_dir/ 下生成 result.md
   - 审计日志无 Critical/High 违规
```
