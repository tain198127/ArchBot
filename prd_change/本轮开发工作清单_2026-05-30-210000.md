# ArchBot Runtime 集成/隔离/分发 — 开发工作清单

> **范围**: 本轮只做 Claude Code / Hermes / OpenCode / OpenClaw 的集成、隔离部署、分发策略
> **不做**: YML 编排引擎（已完成）
> **版本**: v2.0.0-runtime

---

## 一、Rust 后端：Runtime 隔离启动器

| # | 文件 | 做什么 | 状态 |
|---|------|--------|------|
| 1 | `src-tauri/src/agent_runtime/mod.rs` | 注册 agent_runtime 模块 | ☐ |
| 2 | `src-tauri/src/agent_runtime/launcher.rs` | Runtime 隔离启动器：`env_clear()` → 注入隔离 HOME → 白名单环境变量 → 设置工作目录 → spawn 子进程。Windows 上额外注入 `SystemRoot`/`TEMP` | ☐ |
| 3 | `src-tauri/src/agent_runtime/home_setup.rs` | 隔离 HOME 初始化：创建目录 → 按需生成 `.gitconfig` → 按需部署 SSH key（从 Secret Manager 读取） | ☐ |
| 4 | `src-tauri/src/agent_runtime/audit.rs` | Turn 审计：定义 14 条禁止路径规则 + 4 级严重度 → Turn 结束后扫描文件访问 → Critical 终止 / High 告警 | ☐ |

### 这四个文件的关系

```
launcher.rs  ──spawn──▶  Runtime 子进程（隔离 HOME 中运行）
home_setup.rs ──prepare──▶  隔离 HOME 目录（launcher 启动前调用）
audit.rs      ──scan──▶    Turn 结束后检查 Runtime 访问了哪些文件
```

## 五、构建工具：预打包脚本

| # | 文件 | 做什么 | 状态 |
|---|------|--------|------|
| 9 | `runtimes/manifest.yml` | 版本锁文件：4 Runtime × 4 平台 = 16 条记录（version / sha256 / download_url / size） | ☐ |
| 10 | `runtimes/scripts/prepare-claude-code.sh` | `npm pack @anthropic-ai/claude-code-{platform}@{version}` → 提取 native binary → 打包 .tar.gz + 生成 .sha256 | ☐ |
| 11 | `runtimes/scripts/prepare-hermes.sh` | `python3.12 -m venv` → `pip install hermes-agent=={version}` → 打包整个 venv 为 .tar.gz | ☐ |
| 12 | `runtimes/scripts/prepare-opencode.sh` | 从 GitHub Releases 下载对应平台的 binary .tar.gz → 验证 SHA256 → 复制到 cache/ | ☐ |
| 13 | `runtimes/scripts/prepare-openclaw.sh` | `npm pack openclaw@{version}` + 内嵌 Node.js 24 LTS → 打包为 .tar.gz | ☐ |
| 14 | `runtimes/scripts/verify-checksums.sh` | 遍历 cache/ 下所有 .sha256 文件，与 manifest.yml 预期值逐一比对 | ☐ |
| 15 | `runtimes/scripts/prepare-current-platform.sh` | 自动检测当前 OS/架构 → 只下载当前平台的 4 个 Runtime（开发用） | ☐ |
