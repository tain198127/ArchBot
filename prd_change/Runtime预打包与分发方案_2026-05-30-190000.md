## 一、核心原则

> 不在用户机器上执行 `npm install`、`pip install`、`brew install`。
> 四个操作系统各有独立的一套预打包产物，**一份都不能少**。

### 3.2 构建步骤

在 macOS Apple Silicon 机器上执行：

```bash
#!/bin/bash
# scripts/build-all-darwin-arm64.sh
# 在 macOS Apple Silicon 上运行，构建该平台的全部产物

set -euo pipefail
export PLATFORM="darwin-arm64"
export RUST_TARGET="aarch64-apple-darwin"

echo "=== [1/6] 构建 ArchBot.app (Tauri) ==="
cd src-tauri
npm run tauri build -- --target aarch64-apple-darwin
# 产物: src-tauri/target/aarch64-apple-darwin/release/bundle/dmg/ArchBot_2.0.0_aarch64.dmg

echo "=== [2/6] 预打包 Claude Code ==="
bash runtimes/scripts/prepare-claude-code.sh 2.1.128 darwin-arm64
# 过程: npm pack @anthropic-ai/claude-code-darwin-arm64@2.1.128
#   → 提取 native binary
#   → 打包为 claude-code-2.1.128-darwin-arm64.tar.gz

echo "=== [3/6] 预打包 Hermes ==="
bash runtimes/scripts/prepare-hermes.sh 0.14.0 darwin-arm64
# 过程: python3.12 -m venv hermes-venv
#   → pip install hermes-agent==0.14.0
#   → 打包整个 venv 为 hermes-agent-0.14.0-darwin-arm64.tar.gz

echo "=== [4/6] 预打包 OpenCode ==="
bash runtimes/scripts/prepare-opencode.sh 1.15.10 darwin-arm64
# 过程: 从 GitHub Releases 下载 opencode-darwin-arm64.tar.gz
#   → 验证 SHA256 → 复制到 cache/

echo "=== [5/6] 预打包 OpenClaw ==="
bash runtimes/scripts/prepare-openclaw.sh 2026.3.13 darwin-arm64
# 过程: npm pack openclaw@2026.3.13
#   → 内嵌 Node.js 24 LTS for darwin-arm64
#   → 打包为 openclaw-2026.3.13-darwin-arm64.tar.gz

echo "=== [6/6] 交叉编译全部 Adapter ==="
bash adapters/scripts/build-adapters.sh aarch64-apple-darwin
# 过程: 对 4 个 Adapter，分别 cargo build --release --target aarch64-apple-darwin

echo "=== 打包 DMG ==="
bash scripts/package-dmg.sh darwin-arm64
# 过程: 将 ArchBot.app + 4 个 Runtime .tar.gz + 4 个 Adapter .tar.gz
#       打包为 ArchBot-2.0.0-darwin-arm64.dmg

echo "✅ macOS ARM64 全部产物就绪"
```

### 4.3 macOS x64 与 ARM64 的关键差异

| 差异点 | ARM64 | x64 |
|--------|-------|-----|
| Rust target | `aarch64-apple-darwin` | `x86_64-apple-darwin` |
| npm optionalDep | `@anthropic-ai/claude-code-darwin-arm64` | `@anthropic-ai/claude-code-darwin-x64` |
| Python wheel | `*-macosx_11_0_arm64.whl` | `*-macosx_10_9_x86_64.whl` |
| OpenCode binary | `opencode-darwin-arm64` | `opencode-darwin-x64` |
| Node.js (OpenClaw) | `node-v24.14.0-darwin-arm64` | `node-v24.14.0-darwin-x64` |
| DMG 体积 | ~310MB | ~316MB |

**不能交叉**: macOS ARM64 机器上不能构建 x64 的 Python C 扩展（部分 Hermes 依赖含 C 代码），反之亦然。两个 macOS 平台**必须各用一台物理机或 CI runner 独立构建**。

## 七、跨平台构建矩阵（CI 完整配置）

```yaml
# .github/workflows/build-all-platforms.yml
name: Build All Platforms

on:
  push:
    tags: ['v*']
  workflow_dispatch:

jobs:
  # ─────────────────────────────────────────────────
  # macOS Apple Silicon (ARM64)
  # ─────────────────────────────────────────────────
  build-darwin-arm64:
    runs-on: macos-latest    # Apple Silicon M-series
    steps:
      - uses: actions/checkout@v4
      - uses: actions/setup-node@v4
        with: { node-version: '22' }
      - uses: actions/setup-python@v5
        with: { python-version: '3.12' }
      - name: Install Rust
        run: rustup target add aarch64-apple-darwin
      - name: Build All
        run: bash scripts/build-all-darwin-arm64.sh
      - name: Upload DMG
        uses: actions/upload-artifact@v4
        with:
          name: ArchBot-darwin-arm64
          path: dist/ArchBot-2.0.0-darwin-arm64.dmg

  # ─────────────────────────────────────────────────
  # macOS Intel (x64)
  # ─────────────────────────────────────────────────
  build-darwin-x64:
    runs-on: macos-13         # Intel Mac
    steps:
      - uses: actions/checkout@v4
      - uses: actions/setup-node@v4
        with: { node-version: '22' }
      - uses: actions/setup-python@v5
        with: { python-version: '3.12' }
      - name: Install Rust
        run: rustup target add x86_64-apple-darwin
      - name: Build All
        run: bash scripts/build-all-darwin-x64.sh
      - name: Upload DMG
        uses: actions/upload-artifact@v4
        with:
          name: ArchBot-darwin-x64
          path: dist/ArchBot-2.0.0-darwin-x64.dmg

  # ─────────────────────────────────────────────────
  # Windows x64
  # ─────────────────────────────────────────────────
  build-win32-x64:
    runs-on: windows-latest
    steps:
      - uses: actions/checkout@v4
      - uses: actions/setup-node@v4
        with: { node-version: '22' }
      - uses: actions/setup-python@v5
        with: { python-version: '3.12' }
      - name: Install Rust
        run: rustup target add x86_64-pc-windows-msvc
      - name: Build All
        run: bash scripts/build-all-win32-x64.sh
        shell: bash
      - name: Upload MSI
        uses: actions/upload-artifact@v4
        with:
          name: ArchBot-win32-x64
          path: dist/ArchBot-2.0.0-win32-x64.msi

  # ─────────────────────────────────────────────────
  # Linux x64
  # ─────────────────────────────────────────────────
  build-linux-x64:
    runs-on: ubuntu-24.04
    steps:
      - uses: actions/checkout@v4
      - uses: actions/setup-node@v4
        with: { node-version: '22' }
      - uses: actions/setup-python@v5
        with: { python-version: '3.12' }
      - name: Install system deps
        run: |
          sudo apt-get update
          sudo apt-get install -y libgtk-3-dev libwebkit2gtk-4.1-dev libappindicator3-dev
      - name: Install Rust
        run: rustup target add x86_64-unknown-linux-gnu
      - name: Build All
        run: bash scripts/build-all-linux-x64.sh
      - name: Upload AppImage + deb + rpm
        uses: actions/upload-artifact@v4
        with:
          name: ArchBot-linux-x64
          path: |
            dist/ArchBot-2.0.0-linux-x64.AppImage
            dist/archbot_2.0.0_amd64.deb
            dist/archbot-2.0.0-1.x86_64.rpm

  # ─────────────────────────────────────────────────
  # 全平台 SHA256 校验
  # ─────────────────────────────────────────────────
  verify-all:
    needs: [build-darwin-arm64, build-darwin-x64, build-win32-x64, build-linux-x64]
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4
      - name: Download all artifacts
        uses: actions/download-artifact@v4
      - name: Verify all SHA256
        run: bash runtimes/scripts/verify-checksums.sh
```

## 八、产物文件命名规范

```
格式: {name}-{version}-{platform}.{ext}

name:     claude-code | hermes-agent | opencode | openclaw
          archbot-adapter-claude-code | archbot-adapter-hermes | ...
          ArchBot

version:  2.1.128 | 0.14.0 | 1.15.10 | 2026.3.13 | 2.0.0

platform: darwin-arm64 | darwin-x64 | win32-x64 | linux-x64

ext:      .tar.gz (Runtime/Adapter)
          .dmg (macOS)
          .msi (Windows)
          .AppImage | .deb | .rpm (Linux)
```

完整示例：

## 九、本地开发：按需准备当前平台

开发者在 macOS ARM64 上只需准备当前平台：

```bash
# 只准备当前平台（自动检测）
bash runtimes/scripts/prepare-current-platform.sh

# prepare-current-platform.sh 内部:
#   uname -s → Darwin/Linux
#   uname -m → arm64/x86_64
#   映射为 platform 标识：darwin-arm64 / darwin-x64 / linux-x64
#   只下载当前平台的 4 个 Runtime
```

## 十、不做的

- Linux ARM64（市场需求待验证）
- 把二进制 `.tar.gz` 提交到 Git（用 CI artifact / GitHub Releases 托管）
- Docker 化的 Runtime
- 在用户机器上运行 `npm install`、`pip install`、`brew install`
- Universal macOS binary（ARM64 + x64 合并成一个包——两个平台独立分发更清晰）
