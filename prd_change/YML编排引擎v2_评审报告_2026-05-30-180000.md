### 🔴 T1.1 Vue @contextmenu.prevent 与异步 Menu.popup() 的时序冲突

Vue 的 `@contextmenu.prevent` 处理器是同步的——它阻止浏览器默认右键菜单后立即返回。但 Tauri 的 `Menu.popup()` 是异步的（需要跨 JS-Rust 边界创建原生菜单）。

如果处理器返回 Promise 但不 await，菜单可能在事件循环的下一个 tick 才弹出，中间可能出现竞态。如果 await，会阻塞事件处理。

**需要明确**:
- `showContextMenu()` 是否需要 await？如果需要，在 Vue 事件处理器中如何 await？
- Tauri `Menu.popup()` 弹出后，用户点击菜单项 → action 回调 → 这个回调是同步还是异步？回调中的 `ActionRegistry.execute()` 如何处理？

**建议**: Vue 事件处理器中用 `void showContextMenu(...)` 触发但不等待，让 Tauri 的菜单异步弹出。`@contextmenu.prevent` 只负责阻止原生浏览器菜单。

### 🟡 T1.2 permissions.yml 交叉校验规则未细化

提案说"启动时交叉校验，warning 日志"，但未定义校验规则：

```
规则 1: actions.yml 中定义的每个 action id，如果在 context-menus.yml 或 menus.yml 中被引用，
         则该 action id 必须出现在 permissions.yml 的至少一个 allowActions 列表中。
规则 2: permissions.yml 中引用的 action id，必须存在于 actions.yml 中。
规则 3: context-menus.yml 中引用的 action id，必须存在于 actions.yml 中（v1 已有，ConfigValidator）。
```

**建议**: 明确这 3 条校验规则，并在 ConfigValidator 中实现。

### 🟡 T1.3 predicates.yml expression 与 RuntimeState 的接口

`predicates.yml` 中的 `expression` 存储的是表达式字符串（如 `project.loaded == true && context.semanticType == "controller"`）。但 `ExpressionEvaluator.evaluateExpression()` 需要传入 `RuntimeState` 和 `ContextObject`。

当 ContextMenuResolver 需要在过滤菜单项时对 `predicate: xxx` 求值：
1. 从 predicates.yml 查找 id=xxx 的 expression
2. 调用 `evaluateExpression(expression, state, context)`
3. 但此时需要 `RuntimeState` 实例

**需要明确**: ContextMenuResolver 是否需要注入 RuntimeState？还是由调用方（如 FileTreePanel.vue）传入？

### 🔴 T2.1 Tauri MenuItem action 回调缺少权限校验

提案中 Tauri 原生菜单的 action 回调直接调用了 `getActionRegistry().execute()`：

```typescript
action: async () => {
  const params = resolveParams(item.params, context)
  await getActionRegistry().execute(item.action, params, getRuntime())
}
```

但这跳过了 permissions.yml 的 allowActions 校验！第 2 层安全（业务权限）被绕过了。

**修复**: ActionRegistry.execute() 内部统一做 permissions 校验，不依赖调用方的 discipline。所有进入 ActionRegistry 的 execute 调用都必须经过 permissions check。

### 🟡 T3.1 Tauri Menu API 在测试中的 mock 策略

`@tauri-apps/api/menu` 的 `Menu.new()` / `MenuItem.new()` / `Menu.popup()` 在 Node.js 测试环境中不可用（它们依赖 Tauri WebView runtime）。

**建议**: ContextMenuPresenter 不直接 import `@tauri-apps/api/menu`，而是通过依赖注入：

```typescript
interface NativeMenuFactory {
  createMenuItem(opts: {...}): Promise<MenuItemLike>
  createMenu(items: MenuItemLike[]): Promise<MenuLike>
  popup(menu: MenuLike, x: number, y: number): Promise<void>
}
```

桌面环境注入真实的 Tauri 实现，测试环境注入 mock。
