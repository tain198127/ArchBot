## 一、问题陈述

当前 ArchBot 的菜单、右键菜单、动作分发全部硬编码在 TypeScript 中：

| 位置 | 硬编码内容 | 问题 |
|------|-----------|------|
| `src/config/menu.ts` | 6 个菜单分类 + 30+ 子菜单项 | 增改菜单需改 TS 源码 |
| `src/components/layout/MenuBar.vue` | 菜单渲染 + 事件处理混合 | 逻辑与展示耦合 |
| `src/components/domain/FileTreeContextMenu.vue` | 5 个 `v-if type===` 模板 | 每加一种右键场景改组件 |
| `src/composables/useMenuAction.ts` | 字符串 pub/sub 总线 | 无类型安全、无注册机制 |
| `src/App.vue` | 40 行 if-else 动作路由 | 每加一个动作改 App.vue |

**核心痛点**: 增加一个菜单项或右键操作，需要改动 3-5 个文件，且无法在运行时调整。

### 用户场景

**场景 1: 新增功能菜单入口**

> 小明要给 ArchBot 增加"调用链分析"功能，需要在 3 个位置加菜单入口：顶部"分析"菜单下新增子项、右键 Controller 时出现"分析调用链"、右键表时出现"查找涉及该表的接口"。

| | 当前 | 期望 |
|--|------|------|
| 改动文件 | `menu.ts` + `MenuBar.vue` + `FileTreeContextMenu.vue` + `App.vue` | `menus.yml` + `context-menus.yml` + 1 个 action handler |
| 改动行数 | ~25 行 | ~10 行 YML + ~8 行 TS |
| 耗时 | ~15 分钟 | ~5 分钟 |

**场景 2: 为数据标准编辑器增加右键操作**

> 产品要求在 domain 节点右键菜单增加"导出 Excel"功能。

| | 当前 | 期望 |
|--|------|------|
| 改动 | 在 `FileTreeContextMenu.vue` 的 `type==='domain'` 模板中增加一个 `<button>` | 在 `context-menus.yml` 增加 5 行 YML |

### 成功度量

| 度量 | 当前 | 目标 |
|------|------|------|
| 新增菜单项涉及文件数 | 3-5 | 1 (YML) + 可选 1 (actions/*.ts) |
| 新增右键场景涉及文件数 | 1-2 (组件改模板) | 1 (YML) |
| 新增动作涉及文件数 | 2-3 | 1 (actions/*.ts) |
| i18n label | 已通过 key 引用 | 保持，YML 中仍用 i18n key |

## 二、目标架构

```
┌─────────────────────────────────────────────────────────┐
│                    渲染层 (改造)                         │
│  MenuBar.vue          FileTreeContextMenu.vue (简化)     │
│  └─ 从 ConfigLoader    └─ 改为数据驱动，移除 v-if 硬编码 │
│     获取 MenuCategory[]                                   │
├─────────────────────────────────────────────────────────┤
│                    编排层 (新增)                         │
│  ConfigLoader  →  ActionRegistry  →  FlowExecutor       │
│  ContextMenuResolver  →  ExpressionEvaluator            │
│  PredicateRegistry    →  ParameterResolver              │
├─────────────────────────────────────────────────────────┤
│                    配置层 (新增)                         │
│  config/menus.yml    config/context-menus.yml            │
│  config/actions.yml  config/flows.yml                   │
└─────────────────────────────────────────────────────────┘
```

**核心原则**: YML 决定何时、在哪里、按什么顺序调用能力。代码只负责提供受控能力。

## 三、六层安全模型

```
菜单 visibleWhen/enabledWhen  = 用户体验控制  (YML 配置)
Action Registry 白名单校验    = 应用执行控制  (TS 代码)
Tauri Permission              = 系统安全控制  (capabilities/)
Rust Command 参数校验         = 最终资源控制  (Rust 代码)
```

四层缺一不可。菜单隐藏不等于权限控制。

## 五、新增清单

### 5.1 配置层 (`src/config/`)

| 文件 | 内容 | 示例条目 |
|------|------|---------|
| `menus.yml` | 顶部菜单树 (id, label, icon, shortcut, action, children, visibleWhen, enabledWhen) | 文件→打开项目, 配置→场景管理 |
| `context-menus.yml` | 右键菜单 (appliesTo: resourceType, 匹配规则, items[]) | 选中 domain 时显示新建实体/新建字典 |
| `actions.yml` | 动作定义 (id, executor, command/handler, input, result) | project.open → invoke('open_project') |
| `flows.yml` | 多步骤编排 (input, steps[], outputAs, 步骤间数据传递) | 右键 Controller → 扫描→分析→追踪→导出 |

### 5.2 编排层 (`src/orchestration/`)

| 模块 | 职责 |
|------|------|
| `ConfigLoader.ts` | 加载 4 份 YML → 类型化配置对象 → 缓存 |
| `ConfigValidator.ts` | Zod Schema 校验 YML 完整性 |
| `ActionRegistry.ts` | `register(id, handler)` + `execute(id, params, runtime)` |
| `PredicateRegistry.ts` | 复杂条件注册为可测试函数 |
| `ExpressionEvaluator.ts` | 简单表达式求值 (`project.loaded == true`) |
| `ParameterResolver.ts` | 解析 `${context.xxx}` / `${state.xxx}` / `${steps.xxx}` |
| `FlowExecutor.ts` | 串行执行多步骤 Flow，步骤间数据传递 |
| `ContextMenuResolver.ts` | 根据 ContextObject 匹配右键菜单配置 |
| `RuntimeContext.ts` | 聚合当前工程、选中对象、分析结果、用户权限 |

### 5.3 动作注册层 (`src/actions/`)

```
actions/
├── projectActions.ts    # project.open, project.close, project.refresh
├── editorActions.ts     # editor.openFile, editor.format
├── analysisActions.ts   # analysis.* (预留)
├── reportActions.ts     # report.* (预留)
├── uiActions.ts         # ui.refreshPanel, ui.togglePanel
└── dsActions.ts         # ds.newDomain, ds.newEntity, ds.newDict
```

### 5.4 ContextObject 模型

```typescript
interface ContextObject {
  resourceType: 'project' | 'directory' | 'file' | 'domain' | 'entity' | 'dict' | 'table'
  resource: {
    name: string
    path?: string
    [key: string]: unknown
  }
  semanticType?: string  // 语义标注，如 'controller' | 'service'
  attributes?: Record<string, unknown>
}
```

## 七、关键设计决策

### 7.1 i18n 策略

YML 中的 label 字段使用 **i18n key**，而非硬编码中文。渲染时通过 `tt()` 查 i18n：

```yaml
# menus.yml — label 是 i18n key
menus:
  - id: file
    label: menu.file          # → tt('menu.file') → "文件"
    children:
      - id: file.openProject
        label: menuFile.openProject  # → tt('menuFile.openProject') → "打开项目"
```

- 新增菜单项需同步更新 `src/i18n/zh-CN.ts` 和 `src/i18n/en-US.ts`（触发 i18n-sync skill）
- Schema 校验：label 必须是已注册的 i18n key（或至少符合 `^[a-zA-Z][a-zA-Z0-9_.]*$` 格式）

### 7.2 异常降级策略

| 异常场景 | 行为 |
|---------|------|
| `menus.yml` 不存在或 YML 语法错误 | toast 报错 + 降级到**内嵌默认菜单**（从原 `menu.ts` 提取的兜底配置） |
| `actions.yml` 引用未注册的 action id | `ConfigValidator` 启动时报错，拒绝加载，控制台输出冲突 action id |
| `ActionRegistry.register()` 重复 id | throw Error，启动失败 |
| `ExpressionEvaluator` 求值异常 | 表达式语法错误 → 默认返回 `false`（安全侧），日志记录 |
| `FlowExecutor` 第 N 步失败 | 默认 `onError: abort`，停止执行，前序步骤副作用不回滚（第一版无补偿事务） |
| 4 份 YML 全部缺失 | 系统降级到内嵌默认配置，功能可用但不接受新配置 |

### 7.3 交付策略

全量一次交付，15 个新文件 + 4 个退役/改造文件。不再拆分 Phase。

### 7.4 表达式语法约束 (L1 级)

`visibleWhen` / `enabledWhen` 仅支持：

```
运算符: ==  !=  &&  ||  !  ( )
操作数: 布尔字面量 (true/false) | 一级属性访问 (project.loaded, context.resourceType)
禁止: 方法调用、函数调用、.链式二级以上访问、字符串字面量比较
```

示例合法表达式:
```
project.loaded == true
context.resourceType == "domain"
project.loaded == true && context.semanticType == "controller"
```

超出 L1 能力 → 使用 PredicateRegistry 注册复杂条件。

## 八、兼容策略

- 浏览器模式: 纯 Vue 菜单，不使用 Tauri 原生 API
- Tauri 桌面: 仍用 Vue 菜单（保持统一），可通过配置开关切换到原生
- 渐进迁移: ConfigLoader 返回的 `MenuCategory[]` 结构与原 `menu.ts` 兼容，MenuBar.vue 无感切换
- `useMenuAction` 保留但内部委托给 ActionRegistry，现有调用方不报错

## 十、测试策略

### 单元测试

| 模块 | 测试用例 |
|------|---------|
| `ActionRegistry` | 注册成功 / 执行已注册 action / 重复注册抛错 / 执行未注册 id 抛错 |
| `ExpressionEvaluator` | `==` / `!=` / `&&` / `||` / `!` / 括号优先级 / 非法语法返回 false |
| `ParameterResolver` | `${context.resource.path}` / `${state.project.loaded}` / `${steps.xxx}` / 缺失变量返回空串 / 嵌套引用 |
| `ContextMenuResolver` | 精确匹配 `appliesTo` / 无匹配返回空 / 多规则命中取第一条 / 条件过滤 |
| `PredicateRegistry` | 注册/执行/未注册 predicate 抛错 |

### YML Schema 校验测试

| 配置文件 | 合法 case | 非法 case |
|---------|----------|----------|
| `menus.yml` | 含所有必填字段 | 缺 id / label 格式非法 / children 非数组 |
| `context-menus.yml` | appliesTo + items 完整 | appliesTo 缺 resourceType / items 为空 |
| `actions.yml` | 含 executor + command | 缺 id / executor 值非法 |
| `flows.yml` | 含 steps + input | 循环引用 steps / outputAs 重复 |

### 集成测试

- `ConfigLoader` 加载 4 份真实 YML → 返回正确类型化对象
- YML 语法错误 → 降级到内嵌默认菜单 + toast 报错
- `ActionRegistry` 引用未注册 id → 启动时报错

### 覆盖率目标

- `src/orchestration/` ≥ 85%
- `src/actions/` ≥ 70%
