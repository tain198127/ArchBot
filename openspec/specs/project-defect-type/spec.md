## ADDED Requirements

### Requirement: 项目文件树展示缺陷目录

当项目已打开时，系统 SHALL 在文件树的 `projectDirs` 列表中展示"缺陷"子目录项，其位置在"混沌测试"和"部署"之间。

#### Scenario: 打开项目后看到缺陷目录

- **WHEN** 用户在文件菜单中选择"打开项目"并成功打开一个 `.ab` 项目文件
- **THEN** 文件树的子目录列表中显示"缺陷"项
- **AND** "缺陷"项排列在"混沌测试"之后、"部署"之前

### Requirement: 缺陷目录支持国际化

缺陷目录标签 SHALL 根据当前 UI 语言显示对应的翻译文本。

#### Scenario: 中文环境下显示中文标签

- **WHEN** 系统 UI 语言设置为中文
- **THEN** 文件树中的缺陷目录显示为"缺陷"

#### Scenario: 英文环境下显示英文标签

- **WHEN** 系统 UI 语言设置为 English
- **THEN** 文件树中的缺陷目录显示为"Defect"

### Requirement: 缺陷目录功能交互一致性

缺陷目录项 SHALL 拥有和其他项目子目录一致的交互行为（独立颜色、图标、右键菜单、点击跳转）。

#### Scenario: 点击缺陷目录触发导航

- **WHEN** 用户点击文件树中的"缺陷"目录项
- **THEN** 系统发射 `open.defect` 菜单动作（与其他目录的 `open.<key>` 模式一致）
