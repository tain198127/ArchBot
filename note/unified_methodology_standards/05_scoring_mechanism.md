# 05 评价机制与评分体系

## 一、评价对象

评价对象包括：

```text
1. 元能力
2. 组合套路
3. 枚举
4. 关系文件
5. 整体结构
```

---

## 二、元能力评分

| 维度 | 分值 |
|---|---:|
| 复用性 | 15 |
| 输入输出清晰 | 15 |
| 执行语义明确 | 15 |
| 可编排 | 15 |
| 可校验 | 10 |
| 可实现 | 10 |
| 边界清晰 | 10 |
| 不重复 | 10 |
| 总分 | 100 |

判定：

```text
90-100：正式元能力
75-89：候选元能力，需补充定义
60-74：观察项，不进入正式框架
60 以下：不应成为元能力
```

---

## 六、整体结构评分

| 维度 | 分值 |
|---|---:|
| 概念层完整 | 10 |
| 工程化概念完整 | 10 |

判定：

```text

70 以下：不合格，需要重构
```

---

## 七、最终通过标准

统一方法论相关内容只有满足以下条件，才算通过：

```text
1. 所有正式元能力得分 >= 90
2. 所有正式组合套路得分 >= 90
3. 所有必须枚举项得分 >= 85
4. 所有正式关系得分 >= 90
5. 整体结构得分 >= 90
6. 无 P0、P1 未关闭问题
```

---

## 八、P0 / P1 / P2 / P3 判定

### P0 阻断问题

```text
关系方向错误
概念和能力混用
枚举非法值进入 schema
workflow 引用未定义对象
大模型获得流程控制权
无证据结论进入最终产物
```

### P1 高优先级问题

```text
元能力边界不清
组合套路缺少失败路径
枚举值域不稳定
横切能力无映射
关系缺少 evidence
```

### P2 中优先级问题

```text
概念卡不完整
能力卡不完整
组合套路缺少解释器名称
评分缺少证据
```

### P3 优化问题

```text
命名不统一
示例不足
说明不够清晰
文件结构可维护性一般
```
