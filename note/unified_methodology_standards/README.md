# 统一方法论：元能力、组合套路、枚举与关系评估标准包

## 目的

本标准包用于回答四个问题：

```text
1. 什么应该被提炼为元能力？
2. 什么应该被提炼为组合套路？
3. 什么应该被提炼为枚举？
4. 概念、元能力、组合套路、枚举之间的关系应该如何表达和评分？
```

## 文件清单

```text
01_meta_capability_selection_standard.md
  判断一个对象是否应该成为元能力的标准。

02_composition_pattern_selection_standard.md
  判断一个对象是否应该成为组合套路的标准。

03_enum_extraction_standard.md
  判断一个对象是否应该提炼为枚举的标准。

04_relationship_expression_standard.md
  统一方法论中关系文件的表达标准。

05_scoring_mechanism.md
  评价机制与评分体系。

06_relationship_registry_template.yml
  概念、元能力、组合套路、枚举的统一关系表述模板。

07_review_checklist.yml
  可直接用于审查的检查清单。
```

## 总原则

```text
元能力 = 可复用的基础执行 / 认知能力。
组合套路 = 多个元能力在稳定场景下形成的可复用流程模式。
枚举 = 必须被机器稳定判断、路由、校验、阻断或选择的有限值集合。
关系文件 = 描述概念、能力、套路、枚举之间的结构、依赖、控制、校验和生成关系。
```

最短判定公式：

```text
能独立执行、复用、被编排、可输入输出定义的，是元能力。
多个元能力按固定顺序协作解决一类问题的，是组合套路。
需要机器稳定选择、校验、路由、阻断的有限集合，才提炼成枚举。
所有概念、能力、套路、枚举之间的依赖、控制、生成、校验关系都必须进入关系文件。
```
